title: Assistive Vision Glasses
category: hardware
timeframe: "2021"
status: completed
summary: Designed an Ultra-Low-Power Smart Wearable Multisensory Stereo Camera system with AI-enabled capabilities for blind navigation, featuring dual 5MP cameras and onboard neural network processing.
stack:
  - Dual-core ARM STM32 (Cortex M7/M4)
  - Dual 5MP Omnivision cameras
  - Deep neural networks
  - 6D IMU with ML core
  - BLE 5.0
  - OpenCV
  - Embedded vision
impact:
  - Published at IEEE SoutheastCon 2021 with complete open-source design for reproduction by researchers.
  - Achieved time-synchronized stereo capture with onboard object detection and image recognition.
  - Implemented distributed processing architecture with machine learning core offloading decision tree classification.
  - Contributed to open-source blind navigation system development with complete schematic and layout files.
links:
  - label: IEEE
    href: "https://ieeexplore.ieee.org/abstract/document/9401833"
context: "Over 250 million people worldwide live with vision impairment. Existing assistive devices are either too expensive, too bulky, or lack real-time intelligence. This project aimed to create a wearable that could run neural networks onboard — no cloud dependency."
problem: "Blind navigation systems need real-time object detection but are constrained by battery life, weight, and processing power. Cloud-dependent solutions fail without connectivity and introduce latency."
constraints: "Must be ultra-low-power for all-day wear. Must process stereo vision onboard without cloud. Must be reproducible by other researchers (open-source)."
approach: "Used dual-core ARM STM32 with hardware ML core to offload classification. Paired dual 5MP cameras for stereo depth. Implemented distributed processing across cores — one for capture, one for inference."
outcome: "Published at IEEE SoutheastCon 2021. Achieved real-time stereo capture with onboard object detection. All schematics and layout files released as open-source."
lessons: "Embedded ML is about trade-offs — choosing the right model architecture for the hardware matters more than model accuracy on a benchmark. Distributed processing across heterogeneous cores unlocked performance that a single processor couldn't achieve."
